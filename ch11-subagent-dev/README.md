# 第11章: サブエージェント駆動開発

第10章で解説したサブエージェントの基礎（Taskツール、エージェント種別、カスタム定義、権限制御、永続メモリ）を、実際の開発ワークフローに組み込むパターンを7つのレシピで体系化します。

並列調査による高速化、マルチファイルリファクタリング、チェーン方式のワークフロー、バックグラウンド実行、コードレビュー、レガシーコード分析、ドキュメント一括生成まで、サブエージェントを開発の主力として活用する実践テクニックを扱います。

## レシピ一覧

| # | レシピ | 説明 |
|---|--------|------|
| 62 | [並列実行で大規模コードベースを高速調査する](./recipe-62-parallel-explore/) | 複数のExploreサブエージェントを並列起動し、モジュール単位で同時調査する |
| 63 | [マルチファイルリファクタリングの自動化](./recipe-63-multi-file-refactoring/) | Explore（調査）+ general-purpose（実行）の2フェーズでリファクタリングを安全に実行する |
| 64 | [Prompt Chainingと中間ゲート設計](./recipe-64-prompt-chaining/) | サブエージェントを順次呼び出すチェーン方式で、品質を保つワークフローを構築する |
| 65 | [テスト実行とレポート集約（バックグラウンド実行）](./recipe-65-background-test/) | テストをバックグラウンドで実行し、待ち時間に他の作業を並行して進める |
| 66 | [サブエージェント駆動のコードレビュー](./recipe-66-code-review/) | レビュー専用カスタムエージェントと永続メモリで、一貫した品質基準と学習するレビューを実現する |
| 67 | [レガシーコード分析と移行計画の自動生成](./recipe-67-legacy-analysis/) | 並列調査とチェーンを組み合わせ、分析から移行計画生成までを自動化する |
| 68 | [ドキュメント・コメント一括生成](./recipe-68-doc-generation/) | ドキュメント生成専用サブエージェントで、ファイル単位の並列ドキュメント生成を行う |

## 使い方

```bash
# 例: モジュール調査エージェントをプロジェクトに追加
cp -r recipe-62-parallel-explore/.claude /path/to/your-project/

# 例: コードレビューエージェントをプロジェクトに追加
cp -r recipe-66-code-review/.claude /path/to/your-project/

# 例: ドキュメント生成エージェントをプロジェクトに追加
cp -r recipe-68-doc-generation/.claude /path/to/your-project/
```

各エージェント定義ファイルはそのまま`.claude/agents/`にコピーして使えます。プロンプト例は`.txt`ファイルとして提供しているので、プロジェクトに合わせてカスタマイズしてください。
