# レシピ71: 共有タスクリストと依存関係管理

Agent Teamsの共有タスクリストを活用し、タスクの状態管理と依存関係を自動化します。タスクの3つのステータス、依存関係の設定、割り当てパターンを解説します。

## ファイル一覧

| ファイル | 説明 |
|---------|------|
| `prompts/task-list-creation.txt` | 依存関係付きタスクリスト作成のプロンプト例 |
| `prompts/task-management.txt` | タスクの表示・割り当て・確認の操作例 |

## 使い方

```bash
# タスクリストの表示操作
# in-processモード: Ctrl+T でタスクリストの表示/非表示を切り替え

# リーダーにタスク操作を指示
> タスクリストを表示して
> タスク3をバックエンドチームメイトに割り当てて
> タスク2のステータスを確認して
```

## タスクのステータスと依存関係

```
タスクの3つのステータス:
  pending     - 未着手（依存タスクが未完了の場合はブロック状態）
  in_progress - 作業中
  completed   - 完了

依存関係の自動解放:
  タスク1（スキーマ設計）完了
    ├─ タスク2（API実装）が自動的にブロック解除 → claimable
    └─ タスク3（バリデーション）が自動的にブロック解除 → claimable
```

## 割り当て方式

| 方式 | 説明 | 用途 |
|------|------|------|
| リーダー割り当て | リーダーがタスクをチームメイトに直接割り当て | 明確な責任分担が必要な場合 |
| セルフクレーム | チームメイトがタスク完了後に次のタスクを自分で取得 | 柔軟に作業を進めたい場合 |
