# レシピ01: 効果的なCLAUDE.mdの書き方

WHY（プロジェクト概要）、WHAT（ルール・規約）、HOW（コマンド・手順）の3層構造で`CLAUDE.md`を300行以内にまとめる手法です。

## ファイル一覧

| ファイル | 説明 |
|---------|------|
| `CLAUDE.md` | ECサイトバックエンドAPIのCLAUDE.md実例 |

## 使い方

```bash
cp CLAUDE.md /path/to/your-project/CLAUDE.md
```

## カスタマイズ

- **WHYセクション**: プロジェクト名、技術スタック、注意事項を自分のプロジェクトに書き換える
- **WHATセクション**: コーディング規約、ディレクトリ構造をプロジェクトに合わせる
- **HOWセクション**: 開発コマンド、ワークフローを実際の環境に合わせる
- **行数の目安**: `wc -l CLAUDE.md`で300行以内を維持する
