# レシピ08: /compactのカスタム圧縮指示とタイミング戦略

`CLAUDE.md`にcompact指示を記述し、コンテキスト圧縮時に重要な情報を優先的に保持するための設定例です。

## ファイル一覧

| ファイル | 配置先 | 説明 |
|---------|--------|------|
| `CLAUDE.md` | プロジェクトルート | compact指示セクションを含む`CLAUDE.md`の例 |

## 使い方

```bash
# CLAUDE.mdのcompact指示セクションを自分のプロジェクトにマージ
# "Compact instructions" セクションを既存のCLAUDE.mdに追加
```

## カスタマイズ

- **フロントエンド開発**: 「コンポーネント構造とスタイル変更」を重点に
- **バックエンド開発**: 「API仕様とDBスキーマ変更」を重点に
- **デバッグ**: 「エラーメッセージと試行したアプローチ」を重点に
- 圧縮タイミングの目安: コンテキスト使用率70%で手動`/compact`、タスク切り替え時は`/clear`
