# レシピ10: トークン消費を削減するプロンプトテクニック

トークン消費を抑えるための`CLAUDE.md`設計と、プロンプトの書き方の実践例です。

## ファイル一覧

| ファイル | 配置先 | 説明 |
|---------|--------|------|
| `CLAUDE.md` | プロジェクトルート | 簡潔さを重視した`CLAUDE.md`の例（詳細はSkillsに委譲） |

## 主なテクニック

1. **具体的なプロンプト**: 「コードを改善して」ではなく「auth.tsのlogin関数にバリデーションを追加して」
2. **タスク間で/clear**: 無関係なコンテキストの持ち越しを防止
3. **プランモード活用**: `Shift+Tab`で計画を立ててから実装
4. **`CLAUDE.md`の簡潔化**: 500行以下を維持し、詳細はSkillsに移動
5. **CLIツール優先**: MCPサーバーよりgh/aws等のCLIツールを使う

## 使い方

```bash
# CLAUDE.mdを自分のプロジェクトに合わせてカスタマイズ
# 詳細な手順やレビュー指示はSkillsとして分離する
```

## カスタマイズ

- `CLAUDE.md`の行数は`wc -l CLAUDE.md`で定期的に確認（500行以下が目安）
- 使わないMCPサーバーは`/mcp`で無効化し、ツール定義のコンテキスト消費を削減
- 型付き言語ではコード知能プラグイン（Language Server）を導入するとトークン節約効果が大きい
