# レシピ88: コード変更前にGitチェックポイントを作成する

`PreToolUse`フックでファイル編集前にGitの自動コミット（チェックポイント）を作成し、意図しない変更からいつでもロールバックできるようにします。60秒のデバウンスでコミット数の爆発を防止し、作業完了後は`git rebase`でチェックポイントを整理できます。

## ファイル一覧

| ファイル | 説明 |
|---------|------|
| `git-checkpoint.sh` | 編集前にGitチェックポイントを自動作成するスクリプト |
| `hooks-settings.json` | チェックポイントフックの登録用`settings.json`設定例 |
