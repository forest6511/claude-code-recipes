# 機能仕様書の作成

$ARGUMENTS の要件に基づいて、以下の構造で仕様書を作成してください。

## 出力先
specs/{feature-name}.md に保存してください。

## 仕様書の構造
以下のセクションを必ず含めてください:

1. 概要 — 1-2文で機能の目的を説明
2. 背景と動機 — なぜこの機能が必要か
3. 機能要件 — チェックリスト形式
4. 非機能要件 — パフォーマンス基準、セキュリティ考慮事項
5. API設計 — エンドポイント、入出力の定義
6. データモデル — 必要なデータ構造
7. テスト計画 — テストケースのチェックリスト
8. 実装メモ — 注意点、制約

## 手順
1. ユーザーの要件を分析する
2. 既存コードベースとの整合性を確認する
3. 仕様書を作成して保存する
4. 仕様書の内容をユーザーに確認する
